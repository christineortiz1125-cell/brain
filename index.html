How it works

From lens to ears
in seconds.

A six-step pipeline designed to be fast, private, and accessible by default.

1
Select a mode
Choose Read, Simplify, Translate, or Define before you capture.
2
Camera captures
Your Ray-Ban glasses frame the text. Tap to trigger.
3
On-device OCR
Apple Vision or ML Kit extracts text locally. No image leaves the device.
4
Text sent to backend
Only the extracted string travels to the Brain FastAPI server.
5
Claude processes
The backend streams a response from Claude tailored to your mode and profile.
6
Heard aloud
On-device TTS reads the result. Offline fallback serves cached outputs.
Privacy first

Your camera. Your device. Your data.

Brain was designed so that raw images never travel anywhere — ever.

  • OCR runs entirely on-device using Apple Vision (iOS) or ML Kit (Android)
  • Raw images are never transmitted to any server
  • Only the extracted text string reaches the backend
  • Session history is stored in on-device SQLite only — never synced unless you opt in
  • No analytics, no tracking, no ad identifiers
🔒

Offline fallback included

The last 10 session outputs are cached in SQLite. If the backend is unreachable, Brain displays cached processed text rather than failing silently — so you're never left without an answer.
